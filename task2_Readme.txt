Сценарий выполняет резервное копирование журнала логов в назначенную директорию, а также выполняет очистку данной директории от копий старше 7 дней.

Сценарий представлен файлом task2.ps1.
При необходимости возможно изменить следующие параметры:
- путь до исходного лог-файла (переменная src_path),
- каталог назначения резервного копирования (переменная dst_path),
- время хранения копий в днях (переменная max_days).

Для выполнения резервного копирования по расписанию нужно добавить соответствующее задание в планировщик Windows.
Для этого можно использовать прилагающийся скрипт task2_AddToSheduler.ps1, который создаст задание на ежедневное исполнение скрипта резервного копирования в 02:00am. Имя задания: Task2_Backup.

При необходимости возможно изменить следующие параметры скрипта:
- путь до скрипта, подлежащего исполнению (переменная path),
- имя пользователя, от которого будет добавлено задание в планировщик (переменная user).

После создания задания можно посмотреть информацию о нем с помощью команды PS:
Get-ScheduledTask Task2_Backup | Get-ScheduledTaskInfo

Также существует возможность добавить задание с помощью графического интерфейса планировщика.
Запустить планировщик (Taskschd.msc) и импортировать задание из файла Task2_Backup.xml.
